# HalfUpFormat Conversion Specification

## 1. Inbound dependencies
- `java.math.RoundingMode` (HALF_UP constant)
- `java.text.DecimalFormat` — superclass; no Python equivalent
- `java.text.DecimalFormatSymbols` — used to set a custom grouping separator
- `java.text.NumberFormat` — used only as a declared type in `adaptiveFormat`

## 2. Outbound dependents
- Used throughout EPQ for number formatting in reports, tables, and GUI labels.
- `adaptiveFormat` is the primary public API called by reporting code.
- Any EPQ class that constructs `new HalfUpFormat(pattern)` or
  `new HalfUpFormat(pattern, true)` to format individual values.

## 3. Public API surface
| Java constructor / method | Python |
|---|---|
| `HalfUpFormat()` | `__init__(self)` |
| `HalfUpFormat(boolean spaceGrouping)` | `@classmethod from_space_grouping` |
| `HalfUpFormat(String pattern)` | `@classmethod from_pattern` |
| `HalfUpFormat(String pattern, boolean spaceGrouping)` | `@classmethod from_pattern_space_grouping` |
| `HalfUpFormat(String pattern, DecimalFormatSymbols symbols)` | `@classmethod from_pattern_symbols` |
| `format(double number) -> String` | `format(self, number: float) -> str` |
| `static adaptiveFormat(double, int, double) -> String` | `@staticmethod adaptiveFormat` |

`setGroupingSize` and `setGroupingUsed` are also ported (called internally by the
constructors through `super()` chains).

## 4. Overloaded constructors (split plan)
Five constructors → `__init__` + four classmethods (R4).
The no-arg `__init__` covers the `HalfUpFormat()` case.
Each classmethod delegates back to `__init__` after resolving its arguments.

## 5. Mutable-output methods
None.

## 6. External touchpoints
- **`java.text.DecimalFormat`** — no Python equivalent. Reimplemented from scratch
  using `decimal.Decimal` with `ROUND_HALF_UP` context for correct rounding.
  Pattern parsing handles the subset of `DecimalFormat` patterns generated by
  `adaptiveFormat` (fixed and `E`-notation patterns built from `0` and `.` chars).
- **`java.text.DecimalFormatSymbols`** — implemented as a minimal Python shim
  (`DecimalFormatSymbols`) in the same module. Only `setGroupingSeparator` /
  `getGroupingSeparator` and `getInstance()` are needed.
- **`java.text.NumberFormat`** — used only as a declared local type in
  `adaptiveFormat`; replaced by `HalfUpFormat` directly (duck typing, no shim needed).
- **`RoundingMode.HALF_UP`** — `decimal.ROUND_HALF_UP` from Python stdlib.
- **`Math.max`, `Math.abs`, `Math.floor`, `Math.log10`, `Math.pow`, `Math.round`**
  → `max()`, `abs()`, `math.floor`, `math.log10`, `math.pow`, `round()`.

## 7. Suspected Java bugs
None. The class exists specifically to *fix* Java's default HALF_EVEN (banker's)
rounding; HALF_UP is the intended behaviour.

## Implementation notes
Pattern subset handled by `_apply_pattern` / `_format_scientific_halfup`:
- `"0"*n` — integer, no decimal point
- `"0"*m + "." + "0"*n` — fixed-point with n decimal places
- `"0." + "0"*n + "E0"` — scientific notation (generated only inside `adaptiveFormat`)

Grouping applies the MEDIUM MATHEMATICAL SPACE (`U+2006`) as a thousands separator
on the integer part of the formatted result when `spaceGrouping=True`.
